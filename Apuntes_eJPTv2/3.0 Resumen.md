### **Parte 1: Ataques a Sistemas Windows**

#### **1. Visión General de las Vulnerabilidades en Windows**

- **Contexto:** Windows es el SO más utilizado, lo que lo convierte en un objetivo principal. Históricamente ha tenido vulnerabilidades graves (ej. MS08-067, MS17-010).
    
- **Causas Comunes:**
    
    - Desarrollado en C, propenso a desbordamientos de búfer.
        
    - Configuración insegura por defecto.
        
    - Fragmentación del SO: diferentes versiones tienen diferentes vulnerabilidades, y muchos sistemas no están parcheados.
        
    - Vulnerabilidades multiplataforma (ej. inyección SQL).
        
- **Tipos de Vulnerabilidades:**
    
    - Divulgación de información.
        
    - Desbordamientos de búfer.
        
    - Ejecución remota de código.
        
    - Escalada de privilegios.
        
    - Denegación de servicio (DoS).
        
- **Servicios de Windows Frecuentemente Explotados:**
    
    - **Microsoft IIS (puertos 80/443):** Servidor web propietario.
        
    - **WebDAV (puertos 80/443):** Extensión HTTP que permite a los clientes gestionar archivos en un servidor web, actuando como un servidor de archivos.
        
    - **SMB/CIFS (puerto 445):** Protocolo de intercambio de archivos en red local.
        
    - **RDP (puerto 3389):** Protocolo de escritorio remoto.
        
    - **WinRM (puertos 5985/443):** Protocolo de administración remota de Windows.
        

#### **2. Explotación de Servicios Específicos en Windows**

- **Microsoft IIS y WebDAV:**
    
    - **Objetivo:** WebDAV requiere autenticación. Se puede hacer un ataque de fuerza bruta para obtener credenciales legítimas.
        
    - **Herramientas:** `davtest` (para escanear y autenticar) y `cadaver` (cliente WebDAV para subir/descargar archivos).
        
    - **Método:** Con credenciales válidas, se puede subir un payload malicioso (ej. `.asp`) para ejecutar comandos o obtener una reverse shell. Se puede hacer tanto manualmente como con Metasploit.
        
- **SMB con PsExec:**
    
    - **Objetivo:** PsExec es una herramienta de Microsoft que permite ejecutar procesos remotamente. Su autenticación se realiza a través de SMB.
        
    - **Requisito:** Credenciales legítimas (usuario y contraseña/hash), obtenidas mediante fuerza bruta (ej. contra la cuenta `Administrator`).
        
    - **Método:** Con las credenciales, se usa PsExec para autenticarse y ejecutar comandos arbitrarios u obtener una shell.
        
- **MS17-010 (EternalBlue):**
    
    - **Vulnerabilidad:** En el protocolo SMBv1 de Windows. Permite la ejecución remota de código enviando paquetes especialmente diseñados.
        
    - **Origen:** Desarrollada por la NSA y filtrada por el grupo Shadow Brokers. Usada en el ataque de ransomware WannaCry.
        
    - **Sistemas Afectados:** Múltiples versiones de Windows (Vista, 7, Server 2008, 8.1, 10, etc.).
        
    - **Herramientas:** Scripts como `AutoBlue-MS17-010` o módulos de Metasploit.
        
- **RDP (BlueKeep - CVE-2019-0708):**
    
    - **Vulnerabilidad:** En el protocolo RDP. Permite la ejecución remota de código a nivel de kernel sin autenticación.
        
    - **Riesgo:** Puede causar inestabilidad en el sistema. Es crucial usar exploits verificados (como el módulo de Metasploit) para no dañar el objetivo.
        
- **WinRM:**
    
    - **Objetivo:** WinRM permite la administración remota a través de HTTP/S.
        
    - **Herramientas:**
        
        - `crackmapexec`: Para realizar fuerza bruta y ejecutar comandos.
            
        - `evil-winrm`: Para obtener una sesión de shell interactiva.
            

#### **3. Escalada de Privilegios en Windows**

- **Definición:** Proceso de pasar de un usuario con pocos privilegios a otro con más (idealmente Administrador o SYSTEM) tras el acceso inicial.
    
- **Exploits del Kernel de Windows:**
    
    - El kernel (núcleo del SO) tiene control total. Se buscan vulnerabilidades en el kernel para ejecutar código privilegiado.
        
    - **Herramienta:** `Windows-Exploit-Suggester` (compara los parches del sistema con la base de datos de Microsoft para sugerir exploits).
        
- **Bypass de UAC con UACMe:**
    
    - **UAC (User Account Control):** Característica de seguridad que pide permiso para cambios administrativos.
        
    - **Requisito:** Se necesita una cuenta que sea miembro del grupo de administradores locales.
        
    - **UACMe:** Herramienta que abusa de las herramientas "AutoElevate" de Windows para ejecutar payloads con privilegios elevados sin pedir confirmación. Ofrece más de 60 métodos diferentes según la versión de Windows.
        
- **Suplantación de Tokens de Acceso:**
    
    - **Tokens de Windows:** "Llaves temporales" creadas por LSASS al autenticar un usuario. Describen el contexto de seguridad de un proceso.
        
    - **Niveles:**
        
        - **Impersonate-level:** Para logins no interactivos (ej. servicios).
            
        - **Delegate-level:** Para logins interactivos (ej. RDP). Son más peligrosos, ya que pueden suplantarse en otros sistemas.
            
    - **Privilegios Necesarios:** SeRequirePrimaryToken, SeCreateToken, SeImpersonatePrivilege.
        
    - **Módulo Incognito (Metasploit):** Permite listar y suplantar tokens disponibles en el sistema.
        
- **Flujos de Datos Alternativos (ADS):**
    
    - **Concepto:** Atributo de archivos en sistemas NTFS. Cada archivo tiene un flujo de datos (contenido) y un flujo de recursos (metadatos).
        
    - **Técnica:** Los atacantes pueden ocultar código malicioso en el flujo de recursos de un archivo legítimo para evadir la detección de antivirus básicos.
        

#### **4. Dumping de Credenciales en Windows**

- **Base de Datos SAM:**
    
    - Almacena los hashes de las contraseñas de los usuarios. Está bloqueada por el kernel mientras el SO está en ejecución.
        
    - Los atacantes acceden a la memoria del proceso LSASS (que contiene los hashes en caché) para obtenerlos.
        
    - Se requieren privilegios elevados.
        
- **Tipos de Hashes:**
    
    - **LM (LanMan):** Débil, divide la contraseña en dos mitades de 7 caracteres, las convierte a mayúsculas y las hashea por separado con DES. Obsoleto desde Windows Vista.
        
    - **NTLM (NTHash):** Más fuerte, usa MD4, no divide el hash, es sensible a mayúsculas y permite caracteres especiales.
        
- **Archivos de Configuración de Windows:**
    
    - La instalación automatizada de Windows (Unattended Windows Setup) usa archivos como `Unattend.xml` o `Autounattend.xml`.
        
    - Estos archivos pueden contener la contraseña de la cuenta de Administrador (a veces codificada en Base64) y quedar olvidados en el sistema.
        
- **Mimikatz:**
    
    - Herramienta post-explotación muy potente que extrae contraseñas en texto claro, hashes y tickets Kerberos de la memoria (proceso `lsass.exe`).
        
    - En Metasploit, se puede usar la extensión equivalente llamada **Kiwi**.
        

#### **5. Pass-the-Hash (PtH)**

- **Concepto:** Técnica que utiliza el hash NTLM de una contraseña (en lugar de la contraseña en texto claro) para autenticarse en un sistema remoto.
    
- **Herramientas:** Módulo `psexec` de Metasploit, `crackmapexec`.

### **Parte 2: Ataques a Sistemas Linux**

#### **1. Visión General de las Vulnerabilidades en Linux**

- **Contexto:** Linux es un SO de código abierto, comúnmente usado como servidor.
    
- **Servicios de Linux Frecuentemente Explotados:**
    
    - **Apache Web Server (puertos 80/443):** Servidor web dominante.
        
    - **SSH (puerto 22):** Protocolo de acceso remoto seguro.
        
    - **FTP (puerto 21):** Protocolo de transferencia de archivos.
        
    - **SAMBA (puerto 445):** Implementación de Linux del protocolo SMB de Windows.
        

#### **2. Explotación de Servicios Específicos en Linux**

- **Shellshock (CVE-2014-6271):**
    
    - **Vulnerabilidad:** En el shell Bash. Bash ejecuta comandos adicionales después de una secuencia específica de caracteres `() {:;};`.
        
    - **Contexto de Explotación:** En servidores web Apache que ejecutan scripts CGI (`.sh`). Al llamar a un script CGI, el servidor inicia un proceso Bash, lo que permite inyectar comandos maliciosos.
        
- **FTP:**
    
    - **Ataque:** Fuerza bruta para obtener credenciales.
        
    - **Configuración Insegura:** Acceso anónimo habilitado.
        
- **SSH:**
    
    - **Ataque:** Fuerza bruta contra la autenticación por usuario/contraseña para obtener acceso remoto.
        
- **SAMBA:**
    
    - **Ataque:** Fuerza bruta para obtener credenciales.
        
    - **Herramientas:** `SMBMap` (para enumerar, listar y ejecutar comandos) y `smbclient` (cliente con interfaz similar a FTP).
        

#### **3. Escalada de Privilegios en Linux**

- **Exploits del Kernel de Linux:**
    
    - **Metodología:**
        
        1. Identificar la versión del kernel y las vulnerabilidades que le afectan.
            
        2. Descargar, compilar y transferir el exploit al sistema objetivo.
            
    - **Herramienta:** `Linux-Exploit-Suggester` (evalúa el kernel en busca de vulnerabilidades conocidas).
        
- **Trabajos Cron Mal Configurados:**
    
    - **Cron:** Servicio que programa tareas (scripts, comandos) para ejecutarse en un horario específico.
        
    - **Objetivo:** Buscar trabajos cron que se ejecuten como usuario **root**.
        
    - **Técnica:** Si un script ejecutado por root es modificable por nuestro usuario, podemos insertar código malicioso para que, cuando el cron lo ejecute con privilegios de root, nos otorgue una shell con esos mismos privilegios.
        
- **Binarios SUID:**
    
    - **SUID (Set Owner User ID):** Permiso especial en Linux. Cuando se aplica a un binario, cualquier usuario que lo ejecute lo hará con los permisos del **propietario del archivo** (a menudo root).
        
    - **Explotación:** Si un binario con SUID root tiene una vulnerabilidad o puede ser abusado (ej. para lanzar una shell), un usuario sin privilegios puede explotarlo para obtener una shell de root.
        

#### **4. Dumping de Hashes de Contraseñas en Linux**

- **Archivos Clave:**
    
    - `/etc/passwd`: Contiene información de las cuentas de usuario (legible por todos). **NO** contiene las contraseñas.
        
    - `/etc/shadow`: Contiene los hashes de las contraseñas. Solo accesible por el usuario **root**.
        
- **Identificación del Algoritmo de Hash:**
    
    - El prefijo en el campo de la contraseña en `/etc/shadow` indica el algoritmo:
        
        - `$1$` : MD5
            
        - `$2$` : Blowfish
            
        - `$5$` : SHA-256
            
        - `$6$` : SHA-512
