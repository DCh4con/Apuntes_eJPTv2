En sistemas Linux, las contraseñas no se almacenan en texto plano. Se procesan mediante un algoritmo de hashing y se guardan en un archivo protegido al que solo el usuario **root** tiene acceso.

## 📑 1. Identificación de Algoritmos (El archivo `/etc/shadow`)
El archivo `/etc/shadow` contiene los hashes con un formato específico. El primer campo después del nombre de usuario indica el algoritmo utilizado, identificado por un número entre símbolos de dólar (`$id$`).

| **Prefijo** | **Algoritmo**         | **Seguridad Actual**                                 |
| ----------- | --------------------- | ---------------------------------------------------- |
| **\$1\$**   | **MD5**               | Muy bajo (obsoleto, rápido de crackear).             |
| **\$2\$**   | **Blowfish (bcrypt)** | Alto (común en sistemas modernos y bases de datos).  |
| **\$5\$**   | **SHA-256**           | Alto.                                                |
| **\$6\$**   | **SHA-512**           | Muy Alto (estándar actual en la mayoría de distros). |

## 🛠️ 2. Métodos de Extracción
### Método A: Manual (Lectura Directa)
Si tienes una shell como root, simplemente visualiza el contenido del archivo:
- `cat /etc/shadow`

**Acción:** Copia las líneas que te interesen a un archivo en tu máquina atacante (ej: `hashes.txt`) para procesarlas.
### Método B: Metasploit (Automatizado)
Si tienes una sesión de **Meterpreter** o una shell con privilegios dentro de Metasploit, puedes automatizar la recolección.

- **Módulo:** `post/linux/gather/hashdump`
- `set SESSION <ID_SESION_PRIVILEGIADA>` 
- `run`

**Resultado:** Metasploit extraerá los nombres de usuario y sus hashes, guardándolos a menudo en su base de datos local (`creds`).
